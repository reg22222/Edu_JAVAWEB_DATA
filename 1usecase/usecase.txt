use case 자세한 설명
Use Case Diagram은 시스템의 사용자와 시스템 사이의 상호작용을 기술하여 
시스템의 요구사항을 나타낸다. 
Use Case Diagram에서는 use case를 이용하여 시스템의 특정한 사용에 대하여 
기술하며, 복잡한 use case에 대해서는 좀 더 단순한 use case로 세분화하고 
세분화된 use case 간의 의존성을 정의하여 표현할 수 있다. 

그림1, 2 참조하여 만들기
include => 포함(공통적으로 포함하여 사용될때)
여러 유즈케이스에서 중복되는 단계가 있는 경우 이를 떼어 새로운 유즈케이스로 만든다. 
중복을 제거하고 <<include>>라 표기하며 화살표 점선을 사용한다

extends => 확장
특정 조건이 만족되는 경우에 실행되는 단계가 있는 경우 확장관계를 표시한다.
즉, 원래의 루틴에서 따로 확장되서 추가적인 기능을 해야할 때 사용된다.
<<extends>>를 표기하며, 원래의 유스케이스로 돌아올 때는 화살표를 점선으로 표시한다.
포함관계는 반드시 거쳐가야하는 경로지만, 확장관계의 경우에는 특정 경우에만 들어간다.
